# Workflow: UI Package and Storybook

## Goal

Define architecture of the shared UI package using Shadcn, Radix, Tailwind and Storybook.

- design-system-architect
- storybook-specialist
- ui-designer
- frontend-developer
- react-specialist
- typescript-pro
- accessibility-tester
- technical-writer

## Skills

- docs/13-ui-package-storybook/component-testing.md
- docs/13-ui-package-storybook/release-versioning.md

## Rules

No application-specific UI belongs in the UI package.
Every reusable component must define:
- variants
- states
- accessibility behavior
- story examples
- test expectations
