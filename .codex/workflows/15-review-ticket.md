# Workflow: Review Ticket

## Goal

Review one completed ticket before merge.

## Agents

- reviewer
- code-reviewer
- security-auditor
- qa-expert
- accessibility-tester
- performance-engineer

## Skills

- code-review
- security-best-practices
- create-testing-strategy

## Review checklist

- verify acceptance criteria
- verify tests
- verify Storybook coverage for UI components
- verify package boundaries
- verify no undocumented architecture changes
- verify GitHub Issue/Project status is updated

## Outputs

- review summary
- requested changes
- approval or rejection
- ticket status update
