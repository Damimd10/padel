# Labels and Milestones

## Suggested label groups

### Type

- `type:epic`
- `type:story`
- `type:task`
- `type:bug`
- `type:chore`

### Domain

- `area:frontend`
- `area:backend`
- `area:ui-package`
- `area:storybook`
- `area:testing`
- `area:platform`
- `area:docs`

### Affected target

- `target:apps-web`
- `target:apps-api`
- `target:packages-ui`
- `target:packages-api-client`
- `target:packages-schemas`
- `target:packages-config`

### Lane

- `lane:frontend`
- `lane:backend`
- `lane:infrastructure`

### Priority

- `priority:p0`
- `priority:p1`
- `priority:p2`
- `priority:p3`

### Workflow

- `status:blocked`
- `status:needs-docs`
- `status:needs-adr`

## Milestones

Use milestones for:

- sprint buckets, or
- release targets

Choose one convention and keep it consistent across the board.
