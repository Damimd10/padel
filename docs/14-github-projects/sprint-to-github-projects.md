# Sprint to GitHub Projects

## Goal

Translate the approved sprint plan into an executable GitHub Project view.

## Flow

1. finalize `docs/09-agile/sprint-plan.md`
2. confirm each committed item exists in `product-backlog.md`
3. create or update the matching GitHub Issues
4. assign lane label, affected target labels, and sprint label or milestone
5. move selected issues into the correct sprint view/status
6. confirm owners, dependencies and blocked flags
7. confirm frontend/backend/infrastructure lane balance

## Validation checklist

- every sprint item exists in GitHub
- every GitHub sprint item exists in the sprint doc
- no draft/unapproved work is included
- blocked items are visible before sprint start
