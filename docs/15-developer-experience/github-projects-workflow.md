# GitHub Projects Workflow

## Purpose

Define how documentation becomes operational work in GitHub Issues and GitHub Projects.

## Flow

1. docs define scope and decisions
2. approved backlog items are created as Issues
3. Issues are enriched with lane, affected package and workflow metadata
4. sprint selection happens in GitHub Projects
5. implementation starts only from approved sprint tickets

## Rule

GitHub Projects is the operational agile board. Docs remain the source of truth for requirements and architecture.
